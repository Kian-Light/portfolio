To justify threshold selection prior to running the master batch loop, I ran a sensitivity analysis using a 50m buffer against OS Open Rivers. I benchmarked Multiple Flow Direction (MFD) against Single Flow Direction (D8) routing at a 120,000 cell threshold:

Routing Method Threshold (Cells) Buffer Radius Precision (%) Recall (%)
MFD (streams5) 120,000 50m 50.4% 51.0%
D8 (streams_d8) 120,000 50m 50.9% 51.0%
Key Insight: Changing the routing algorithm from MFD to D8 provided a negligible precision bump (50.9% vs 50.4%). The inherent ceiling limit (~51% recall) points to differences between purely topographic DEM generation and ground-surveyed river vectors, rather than threshold selection.

2. Custom Sensitivity Evaluation Functions
# Compute exact precision metrics inside spatial buffers
score_threshold <- function(streams_raster, buffer_vect) {
  # Convert stream extraction cells to distinct vector points
  stream_pts <- as.points(streams_raster)
  total_cells <- length(stream_pts)
  
  # Check intersection matrices inside the validation boundary
  inside <- is.related(stream_pts, buffer_vect, "intersects")
  matched_cells <- sum(inside)
  pct_match <- 100 * matched_cells / total_cells
  
  return(list(
    total = total_cells, 
    matched = matched_cells, 
    pct_match = pct_match
  ))
}

# Calculate recall metrics against the source river network lines
score_recall <- function(streams_raster, rivers_line, buffer_dist = 50) {
  stream_pts <- as.points(streams_raster)
  stream_buffer <- st_buffer(st_as_sf(stream_pts), dist = buffer_dist)
  stream_buffer_union <- st_union(stream_buffer)
  
  rivers_covered <- st_intersection(rivers_line, stream_buffer_union)
  pct_length_covered <- 100 * sum(st_length(rivers_covered)) / sum(st_length(rivers_line))
  
  return(pct_length_covered)
}
